# Project Rules - 项目规则文档

本目录存放所有项目的规则文档。每个项目一个文件，记录该项目的重要规范、访问规则、技术栈等信息。

## 📁 文件命名规范

```
{项目名}-项目规则.md
```

例如：
- `茂名文旅-项目规则.md`
- `润德教育-项目规则.md`
- `山西培训平台-项目规则.md`

## 📝 规则文档应包含什么

### 1. 访问规范（必须）
- 如何正确访问开发/测试环境
- 禁止的操作（如直接访问容器端口）
- Nginx/Docker 配置说明

### 2. 项目信息
- 项目路径
- 技术栈
- 主要目录结构

### 3. 重要规则
- 环境访问限制
- 数据操作规范
- 安全注意事项

### 4. 历史记录
- 已完成的工作
- 已知问题和解决方案
- 常见错误及修复方法

## 🔍 如何使用

**主 Agent (小d) 工作流程：**

1. 当用户提到某个项目时，先读取对应的项目规则文档
2. 按照规则文档中的规范执行操作
3. 如有新发现，更新规则文档

**示例：**
```
用户：处理一下茂名文旅的图片上传问题

主 Agent：
1. 读取 ~/clawd/project-rules/茂名文旅-项目规则.md
2. 了解访问规范（必须通过Nginx）
3. 查看项目路径和技术栈
4. 开始处理问题
```

## ➕ 添加新项目

1. 创建新文件：`{项目名}-项目规则.md`
2. 填写项目规范（参考现有文件格式）
3. 在 MEMORY.md 中添加项目概览
4. 更新本 README.md 的表格

---

**注意：** 项目规则文档是重要的知识沉淀，每次发现新的项目规范都应更新对应文档。
