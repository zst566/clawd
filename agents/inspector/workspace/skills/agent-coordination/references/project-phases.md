# Project Phase Checklists

## Code Development Projects

### Phase 1: Design Documentation
- [ ] Component breakdown plan
- [ ] Composables extraction plan
- [ ] State management design
- [ ] Key data flow documentation

### Phase 2: Backend Implementation
- [ ] Directory structure created
- [ ] Controllers split
- [ ] Services split
- [ ] Routes updated

### Phase 3: Backend Code Review
- [ ] Security review report (@guardian)
- [ ] Quality review report (@inspector)
- [ ] Issues fixed
- [ ] Re-review passed

### Phase 4: Frontend Design
- [ ] Component list with props/emits
- [ ] Composables list with functions
- [ ] State management plan

### Phase 5: Frontend Implementation
- [ ] Directory structure created
- [ ] All components implemented
- [ ] All composables implemented
- [ ] Main entry file refactored

### Phase 6: Frontend Code Review
- [ ] Security review report (@guardian)
- [ ] Quality review report (@inspector)
- [ ] Issues fixed
- [ ] Re-review passed

### Phase 7: Testing
- [ ] Test cases designed
- [ ] Tests executed
- [ ] Bugs fixed

### Phase 8: Acceptance
- [ ] Feature verification
- [ ] Performance verification
- [ ] Documentation updated

## Data Analysis Projects

### Phase 1: Data Understanding
- [ ] Data schema analyzed
- [ ] Data quality assessment
- [ ] Analysis plan

### Phase 2: Data Processing
- [ ] Scripts executed
- [ ] Results generated
- [ ] Anomalies identified

### Phase 3: Report Generation
- [ ] Analysis report
- [ ] Visualizations
- [ ] Recommendations

## General Rule

**NEVER** start Phase N+1 before Phase N is confirmed complete.
