## 二、发现的问题

### 🔴 严重问题

#### 1. 代码复用性极差 - 大量重复代码

**问题描述**:  
三个登录页面的代码几乎完全相同，仅 `clientType` 参数和跳转页面不同。

| 端 | clientType | 跳转页面 |
|---|---|---|
| h5-client | `user` | `home-new` |
| h5-channel | `channel` | `home` |
| h5-teacher | `teacher` | `home` |

**建议**: 提取公共登录组件 `LoginPage.vue`，通过 props 传递差异配置。

---

#### 2. 生产环境暴露测试账号信息

**问题描述**:  
三个页面都包含开发环境测试账号信息：

```vue
<div class="dev-tip">
  <p>开发环境测试账号</p>
  <p>用户名: test_user</p>
  <p>密码: 123456</p>
</div>
```

**风险**: 生产环境部署时会被用户看到，造成安全隐患。

**建议**: 
```vue
<div class="dev-tip" v-if="isDevEnv">
  ...
</div>
```
或使用环境变量控制显示。

---

### 🟡 中等问题

#### 3. 类型定义不一致

| 文件 | 返回类型 |
|---|---|
| h5-client/src/api/auth.ts | `Promise<ApiResponse<LoginResponse>>` |
| h5-channel/src/api/auth.ts | `Promise<any>` |
| h5-teacher/src/api/auth.ts | `Promise<any>` |

**影响**: h5-channel 和 h5-teacher 失去类型安全保障。

**建议**: 统一使用 `ApiResponse<LoginResponse>` 类型。

---

#### 4. API 导入不一致

- h5-client 导入了 `ApiResponse` 类型
- h5-channel 和 h5-teacher 未导入

**建议**: 统一导入类型定义。

---

### 🟢 轻微问题

#### 5. 登录失败错误信息不够友好

```typescript
} catch (error: any) {
  showToast({
    message: error.message || '登录失败，请检查网络',
    type: 'fail',
  })
}
```

**问题**: `error.message` 可能不存在，应先检查 error 结构。

**建议**:
```typescript
} catch (error: any) {
  const message = error?.response?.data?.message || error?.message || '登录失败，请检查网络'
  showToast({ message, type: 'fail' })
}
```

---

#### 6. 缺少输入验证

- 未限制用户名/密码长度
- 未过滤特殊字符（虽然 Vue 默认转义）

**建议**: 添加长度和格式验证规则。
