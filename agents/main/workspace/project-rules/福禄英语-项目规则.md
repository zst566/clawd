## ⚠️ 重要：多容器架构访问规范

### 必须使用 Nginx 统一入口

**188 服务器是多容器微服务架构，所有访问必须通过 Nginx 反向代理！**

```
用户请求 → Nginx (8888端口) → 分发到各容器
              ↓
    ┌─────────┼─────────┐
    ↓         ↓         ↓
 C端H5    渠道端    教师端   ...
```

### 正确的访问方式

| 服务 | 正确访问地址 | ❌ 禁止直接访问 |
|------|-------------|----------------|
| C端 H5 | http://192.168.31.188:8888/ | ~~http://192.168.31.188:5173~~ |
| 渠道端 H5 | http://192.168.31.188:8888/channel/ | ~~http://192.168.31.188:5174~~ |
| 教师端 H5 | http://192.168.31.188:8888/teacher/ | ~~http://192.168.31.188:5175~~ |
| PC管理端 | http://192.168.31.188:8888/admin/ | ~~http://192.168.31.188:5176~~ |
| API 网关 | http://192.168.31.188:8888/api/ | ~~http://192.168.31.188:3000~~ |

### 为什么不能直接访问容器端口？

1. **跨域问题** - 前后端分离架构需要统一域名
2. **Cookie/Session** - 认证信息需要在同一域名下传递
3. **API 路由** - 前端请求 `/api/*` 需要 Nginx 代理到后端
4. **安全性** - 暴露容器端口增加攻击面
5. **负载均衡** - 未来可扩展多实例

### 开发调试例外

**仅开发调试时可短暂访问容器端口**（如查看 Vite 热重载日志）：
```
http://192.168.31.188:5173  # C端H5 开发服务器（仅限调试）
```

**但测试功能、API 请求、登录态验证必须使用 8888 端口！**

## 🧪 测试环境规范（重要！）

### 必须使用 188 服务器 Docker 环境

**⚠️ 禁止在本地启动开发服务器进行测试！**

所有测试工作必须通过 188 服务器的 Docker 容器环境进行：

| 服务 | 容器名称 | 访问地址 |
|------|----------|----------|
| C端 H5 | fulu-english-h5-client | http://192.168.31.188:8888 |
| 渠道端 H5 | fulu-english-h5-channel | http://192.168.31.188:8888/channel/ |
| 教师端 H5 | fulu-english-h5-teacher | http://192.168.31.188:8888/teacher/ |
| PC管理端 | fulu-english-pc-admin | http://192.168.31.188:8888/admin/ |
| API 网关 | fulu-english-backend | http://192.168.31.188:8888/api/ |

### 正确测试流程

1. **确保 188 服务器容器已启动**
   ```bash
   ssh zhou@192.168.31.188
   cd /opt/fulu-english
   docker compose ps
   ```

2. **使用 browser 工具访问并检测页面状态**
   ```
   browser open url=http://192.168.31.188:8888
   ```

3. **检测 502 错误的方法**
   
   **方法1：截图检查**（推荐）
   ```
   browser screenshot
   ```
   - 502 错误会在截图中显示 "502 Bad Gateway"
   
   **方法2：控制台检查**
   ```
   browser console
   ```
   - 查看是否有网络请求失败或 CORS 错误
   
   **方法3：JavaScript 执行检查**
   ```
   browser evaluate fn="() => ({ title: document.title, status: document.readyState, has502: document.body?.innerText?.includes('502') })"
   ```
   - 检查页面内容是否包含 "502" 字样
   
   **方法4：网络请求监控**
   ```
   browser evaluate fn="() => performance.getEntriesByType('resource').filter(r => r.responseStatus >= 400).map(r => ({url: r.name, status: r.responseStatus}))"
   ```
   - 检查是否有失败的资源请求

### 502 错误排查流程

**当 browser 工具检测到 502 错误时：**

1. **检查 Nginx 容器日志**
   ```bash
   ssh zhou@192.168.31.188
   docker logs fulu-english-nginx --tail 50
   ```

2. **检查后端服务状态**
   ```bash
   docker ps | grep fulu-english
   docker logs fulu-english-backend --tail 50
   ```

3. **检查容器健康状态**
   ```bash
   docker inspect --format='{{.State.Status}} {{.State.Health.Status}}' fulu-english-backend
   ```

4. **重启异常服务**
   ```bash
   cd /opt/fulu-english
   docker compose restart backend
   # 或重启全部
   docker compose restart
   ```

# 或
node server.js
# 或
