## 福禄英语开发 - 测试方法改进

### 新测试方法：自然语言驱动 Kimi CI 进行 CRUD 测试

**背景**：在福禄英语项目 PC 端开发中，总结出一种高效的测试方法。

**具体做法**：
1. 检查 PC 端页面是否报错
2. 如果页面无报错，直接通过自然语言让 Kimi CI 测试该模块的 CRUD 功能：
   - 新增操作（如：新增校区）
   - 编辑操作
   - 删除操作

**优势**：
- 更高效地发现问题和边界情况
- 避免手动重复点击多次
- 减少重复输入
- Kimi CI 可以快速遍历各种场景

**适用场景**：PC 端模块功能测试，特别是增删改查类操作

## 福禄英语 - Prisma 更新与部署规范

**记录位置**：项目 agents 文档

**关键规则**：
1. **何时需要更新 Prisma**：修改了 `schema.prisma` 文件后
2. **何时需要重新部署**：需要对容器进行重启重构的时候
3. **Agent 自动执行**：在满足上述条件时，Agent 需要自动执行以下操作：
   - 运行 `npx prisma generate`
   - 运行 `npx prisma db push` 或 migration
   - 重启相关容器服务
