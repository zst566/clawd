# 多智能体并发任务分配方案 - 票根优惠模块后端开发

## 原则

1. **使用 `sessions_spawn` 分配所有任务**
   - 短期任务（<30分钟）：使用默认超时（300秒）
   - 长期任务（1-2小时）：设置 `runTimeoutSeconds: 7200`
   - 超长任务（如完整开发）：分段分配，每段2小时

2. **并发执行**
   - 无依赖的任务同时启动
   - 有依赖的任务串行等待

3. **提前完成响应机制**
   - 每个子任务完成后自动汇报
   - 我监控子任务状态，提前完成时立即响应
   - 自动触发下一阶段

## 提前完成响应机制

### 方案：监控+自动触发

```javascript
// 1. 启动所有并发子任务
const tasks = await Promise.all([
  sessions_spawn({ agentId: "codecraft", task: "任务A", runTimeoutSeconds: 7200 }),
  sessions_spawn({ agentId: "codecraft", task: "任务B", runTimeoutSeconds: 7200 }),
  sessions_spawn({ agentId: "codecraft", task: "任务C", runTimeoutSeconds: 7200 })
]);

// 2. 监控子任务状态（每10分钟检查）
// 如果提前完成，立即响应并分配下一阶段

// 3. 自动触发下一阶段
if (allTasksCompleted) {
  sessions_spawn({ agentId: "codecraft", task: "下一阶段任务", ... });
}
```

### 提前完成处理流程

1. 子任务完成 → 自动发送完成报告
2. 我检测到完成 → 立即确认结果
3. 如满足条件 → 自动触发下一阶段
4. 在群组汇报进度
