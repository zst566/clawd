## 🔴 必须修复的高风险/高优先级问题

### 1. 敏感数据明文存储（🔴 安全-高）
**问题**: 手机号、openid 明文存储  
**风险**: 隐私合规风险、微信数据合规风险  
**修复**: 
```javascript
// 应用层加密存储
phone: encrypt(phone, ENCRYPTION_KEY)
```

### 2. 核销码生成算法不明确（🔴 安全-高）
**问题**: 文档未明确随机码生成算法  
**风险**: 如果使用 Math.random() 可能被暴力破解  
**修复**:
```javascript
const crypto = require('crypto');
const randomIndex = crypto.randomInt(0, chars.length);
```

### 3. 撤销核销权限验证不足（🔴 安全-高）
**问题**: 店员可撤销其他店员/店长的核销记录  
**修复**: 增加权限验证逻辑

### 4. API 版本控制缺失（🔴 架构-高）
**问题**: 接口无版本控制  
**影响**: 未来 API 升级导致兼容性问题  
**修复**:
```
GET /api/v1/ticket/types
GET /api/v1/merchant/verify/check
```

### 5. 距离计算实现未详述（🔴 架构-高）
**问题**: 商户列表按距离排序的方案未说明  
**修复**: 使用 MySQL `ST_Distance_Sphere` 计算距离

### 6. OCR 异步处理方案缺失（🔴 架构-高）
**问题**: OCR/AI 识别可能耗时 2-10 秒，当前为同步 API  
**修复**: 任务队列 + WebSocket 推送，或轮询接口

### 7. 核销码高并发唯一性（🔴 架构-高）
**问题**: 随机码有碰撞风险  
**修复**: Redis INCR + 格式化，或预生成码池

### 8. 撤销核销时间限制漏洞（🟡 安全-中）
**问题**: 10分钟限制存在时钟偏移、并发问题  
**修复**: 使用后端服务器时间，增加并发控制
