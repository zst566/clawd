# 与码匠（@zhou_codecraft_bot）通讯规范

## 通讯原则

1. **直接 @ 优先**：在 Telegram 群组中直接 @zhou_codecraft_bot 分配任务
2. **进度检查**：每5分钟主动检查一次进度，不能等待对方汇报
3. **主动推进**：不要等待码匠请示，主动推进到下一阶段
4. **明确交付物**：每次分配任务必须明确交付物、截止时间、验收标准

## 进度检查话术模板

### 首次分配任务
```

**任务范围**：
1. ...
2. ...

**交付标准**：
- ...
- ...

**截止时间**：[时间]

请确认收到并给出预计完成时间！
```

### 5分钟进度检查
```
@zhou_codecraft_bot 进度检查：

1. 当前完成百分比？
2. 是否有阻塞问题？
3. 能否按时完成？

请立即回复！
```

### 阶段完成确认
```
@zhou_codecraft_bot [阶段X] 是否已完成？

请确认以下交付物：
- [ ] 交付物1
- [ ] 交付物2

如已完成，立即开始 [阶段X+1]！
```

## 阻塞处理流程

1. **10分钟无响应**：再次 @ 提醒
2. **20分钟无响应**：标记阻塞，通知项目经理
3. **发现技术阻塞**：立即升级，寻求解决方案

## 常见错误纠正

| 错误 | 正确做法 |
|------|----------|
| 等待码匠主动汇报 | 每5分钟主动检查 |
| 混淆阶段4和阶段5 | 阶段4完成后再进入阶段5 |
| 接受模糊的进度回复 | 要求具体的百分比和交付物状态 |
| 允许无限期延迟 | 设定明确的截止时间 |
