# 任务完成时间统计与超时优化机制

## 问题
当前 `sessions_spawn` 默认超时时间（2小时）基于经验设定，缺乏数据支撑。

## 解决方案：建立任务完成时间统计

### 1. 数据收集

每次使用 `sessions_spawn` 分配任务时，记录以下信息：

```javascript
const taskMetrics = {
  taskId: "runId",
  label: "任务标识",
  taskType: "数据库配置|API开发|测试|...",
  estimatedTime: "预估时间",
  actualTime: "实际完成时间",
  timeoutSetting: "设置的超市时间",
  completedBeforeTimeout: "是否超时前完成",
  complexity: "简单|中等|复杂"
};
```

### 2. 统计维度

| 维度 | 说明 |
|------|------|
| 任务类型 | 数据库设计、API开发、前端组件、测试等 |
| 复杂度 | 简单（<1h）、中等（1-3h）、复杂（>3h） |
| 实际耗时 | 从分配到完成的实际时间 |
| 超时率 | 超过设定时间的任务比例 |

### 3. 动态超时建议

基于统计数据，动态调整默认超时：

```javascript
// 示例统计结果（需要积累至少10个样本）
const timeoutRecommendations = {
  "数据库设计": {
    average: "45分钟",
    p90: "90分钟",
    recommended: "2小时"
  },
  "API开发-简单": {
    average: "30分钟", 
    p90: "60分钟",
    recommended: "1.5小时"
  },
  "API开发-中等": {
    average: "90分钟",
    p90: "3小时", 
    recommended: "4小时"
  },
  "前端组件": {
    average: "60分钟",
    p90: "2小时",
    recommended: "3小时"
  }
};
```

### 4. 实施步骤

**阶段1：数据收集（当前开始）**
- [x] 创建统计记录文件
- [ ] 每次任务完成后记录实际耗时

**阶段2：分析优化（1周后）**
- [ ] 分析统计数据
- [ ] 计算平均耗时、P90分位数

**阶段3：动态调整（持续）**
- [ ] 每月更新统计数据
- [ ] 根据任务类型自动推荐超时时间
