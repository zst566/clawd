# 多智能体任务监督机制

## 问题
当前问题：被动等待子任务完成汇报，可能导致静默时间过长

## 解决方案：主动监督机制

### 1. 定时检查策略

| 任务类型 | 检查频率 | 检查方式 |
|---------|---------|---------|
| 短任务（<30分钟） | 每10分钟 | 主动询问进度 |
| 常规任务（1-2小时） | 每15分钟 | 主动询问进度 |
| 长任务（>2小时） | 每30分钟 | 主动询问进度 |

### 2. 检查内容

每次检查询问：
1. 当前完成百分比？
2. 是否有阻塞问题？
3. 预计多久完成？
4. 是否需要协助？

### 3. 静默检测

**静默定义**：超过检查周期无响应

**处理流程**：
1. 第一次静默 → 主动询问
2. 第二次静默（+10分钟）→ 标记关注
3. 第三次静默（+20分钟）→ 标记阻塞，通知用户

### 4. 提前完成检测

子任务完成自动汇报 → 立即确认 → 检查是否全部完成 → 触发下一阶段

### 5. 超时处理

**超时定义**：超过设置的超时时间

**处理流程**：
1. 检查实际进度
2. 如接近完成 → 延长超时
3. 如阻塞 → 询问原因
4. 必要时重新分配

## 实施方式

### 方案A：Cron定时任务（推荐）

```javascript
// 添加检查任务到Cron
cron.add({
  name: 'task-monitor-ticket-backend',
  schedule: { kind: 'every', everyMs: 15 * 60 * 1000 }, // 每15分钟
  payload: {
    kind: 'agentTurn',
    message: '检查票根模块开发进度，询问码匠当前状态'
  }
});
```

### 方案B：Heartbeat检查

在HEARTBEAT.md中添加检查清单：
```markdown
## 票根模块开发监督
- [ ] 检查阶段3子任务状态
- [ ] 询问码匠进度
- [ ] 确认是否有阻塞
```

### 方案C：Subagent监督器

```javascript
// 启动一个专门的监督子任务
sessions_spawn({
  agentId: "main",
  label: "task-monitor",
  runTimeoutSeconds: 86400, // 24小时
  task: `
    监督票根模块开发进度：
    1. 每15分钟检查所有子任务状态
    2. 询问码匠进度
    3. 发现阻塞立即汇报
    4. 阶段完成后触发下一阶段
  `
});
```
