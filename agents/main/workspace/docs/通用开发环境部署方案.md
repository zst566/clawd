## 核心原则

### 1. 服务器端构建

**为什么不在本地构建？**
- 本地 Mac (ARM64) → 服务器 Linux (AMD64) 跨平台兼容性问题
- 避免传输 node_modules（文件多、体积大）
- 利用服务器更强的性能进行构建

**构建流程**：
```
本机代码 ──rsync──→ 服务器构建区 ──build──→ Docker 镜像 ──run──→ 容器
```

### 2. 双目录结构

| 目录 | 用途 | 内容 |
|------|------|------|
| `/opt/{project}-build/` | 代码构建 | 源代码、node_modules、构建产物 |
| `/opt/{project}/` | 运行配置 | docker-compose.yml、MySQL数据、Nginx配置 |

**分离原因**：
- 构建区可以频繁清理/重建
- 运行区保持数据持久化（MySQL数据、上传文件）

### 3. 统一入口

**Nginx 反向代理**（唯一对外暴露80端口）：
```
http://192.168.31.188/              → h5-client (C端)
http://192.168.31.188/channel/      → h5-channel (渠道端)
http://192.168.31.188/teacher/      → h5-teacher (教师端)
http://192.168.31.188/admin/        → pc-admin (管理端)
http://192.168.31.188/api/          → backend (API)
```

## 文件清单

每个项目需要创建以下文件：

### 1. Docker Compose 配置

**文件**：`docker-compose.188.yml`

```yaml
version: '3.8'

services:
  mysql:
    image: mysql:8.0
    container_name: {project}-mysql
    ports:
      - "3306:3306"
    volumes:
      - ./mysql/data:/var/lib/mysql
      - ./mysql/init:/docker-entrypoint-initdb.d:ro
    environment:
      - MYSQL_ROOT_PASSWORD=${MYSQL_ROOT_PASSWORD}
      - MYSQL_DATABASE={project}_dev
    networks:
      - {project}-network
    restart: always

  backend:
    image: {project}-backend:latest
    container_name: {project}-backend
    expose:
      - "3000"
    environment:
      - DATABASE_URL=mysql://root:${MYSQL_ROOT_PASSWORD}@mysql:3306/{project}_dev
      - JWT_SECRET=${JWT_SECRET}
    depends_on:
      - mysql
    networks:
      - {project}-network
    restart: always

  h5-client:
    image: {project}-h5-client:latest
    container_name: {project}-h5-client
    expose:
      - "80"
    depends_on:
      - backend
    networks:
      - {project}-network
    restart: always

  nginx:
    image: {project}-nginx:latest
    container_name: {project}-nginx
    ports:
      - "80:80"
    volumes:
      - ./nginx/conf.d:/etc/nginx/conf.d:ro
    depends_on:
      - backend
      - h5-client
    networks:
      - {project}-network
    restart: always

networks:
  {project}-network:
    driver: bridge
```

### 2. 部署脚本

**文件**：`deploy188.sh`

```bash
#!/bin/bash
# {project} - 188服务器部署脚本

set -e

SERVER_USER="zhou"
SERVER_IP="192.168.31.188"
SERVER="${SERVER_USER}@${SERVER_IP}"
PROJECT_DIR="/Volumes/.../{project}"
REMOTE_DIR="/opt/{project}-build"
RUN_DIR="/opt/{project}"

# 步骤1: 同步代码到服务器
echo "📦 同步代码到服务器..."
rsync -avz --delete \
  --exclude='.git' \
  --exclude='node_modules' \
  --exclude='dist' \
  "$PROJECT_DIR/" "$SERVER:$REMOTE_DIR/"

# 步骤2: 在服务器上构建镜像
echo "🔨 服务器端构建..."
ssh $SERVER << EOF
  cd $REMOTE_DIR/apps/h5-client
  npm install --registry=https://registry.npmmirror.com
  npm run build
  docker build --platform linux/amd64 -t {project}-h5-client:latest .
  
  cd $REMOTE_DIR/apps/backend
  docker build --platform linux/amd64 -t {project}-backend:latest .
EOF

# 步骤3: 同步配置文件
echo "📄 同步配置文件..."
scp docker-compose.188.yml $SERVER:$RUN_DIR/docker-compose.yml
ssh $SERVER "mkdir -p $RUN_DIR/mysql/data $RUN_DIR/nginx/conf.d"

# 步骤4: 启动服务
echo "🔄 启动服务..."
ssh $SERVER << EOF
  cd $RUN_DIR
  docker compose down 2>/dev/null || true
  docker compose up -d
EOF

echo "✅ 部署完成！访问 http://$SERVER_IP/"
```

### 3. Nginx 配置

**文件**：`nginx/conf.d/188.conf`

```nginx
server {
    listen 80;
    server_name _;

    # C端H5
    location / {
        proxy_pass http://{project}-h5-client:80;
    }

    # API
    location /api/ {
        proxy_pass http://{project}-backend:3000/;
        proxy_http_version 1.1;
    }
}
```

### 4. Dockerfile（前端）

**文件**：`apps/h5-client/Dockerfile`

```dockerfile
# 构建阶段
FROM --platform=linux/amd64 node:20 AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

# 生产阶段
FROM --platform=linux/amd64 nginx:latest
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

### 5. Dockerfile（后端）

**文件**：`apps/backend/Dockerfile`

```dockerfile
FROM --platform=linux/amd64 node:20
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build
EXPOSE 3000
CMD ["node", "dist/main.js"]
```

### 6. 环境变量文件

**文件**：`.env`（不提交到Git）

```
MYSQL_ROOT_PASSWORD=your_password
JWT_SECRET=your_secret
```

## 两种同步模式

### 模式A：实时开发同步（推荐开发阶段使用）

**脚本**：`sync.sh`

**适用场景**：
- 开发阶段频繁修改代码
- 需要立即在服务器上看到效果
- 调试样式、调整布局

**机制**：
```
本地修改代码
    ↓
fswatch 检测到文件变化（1秒内）
    ↓
自动 rsync 同步到服务器 ~/projects/
    ↓
在服务器上手动/自动构建查看效果
```

**文件**：`sync.sh`

```bash
#!/bin/bash
# 项目实时同步脚本

LOCAL_DIR="$(cd "$(dirname "$0")" && pwd)"
PROJECT_NAME=$(basename "$LOCAL_DIR")
REMOTE_HOST="zhou@192.168.31.188"
REMOTE_DIR="~/projects/$PROJECT_NAME"

echo "🚀 启动实时同步: $LOCAL_DIR → $REMOTE_HOST:$REMOTE_DIR"

# 首次全量同步
rsync -avz --delete \
  --exclude='.git' \
  --exclude='node_modules' \
  --exclude='.DS_Store' \
  --exclude='dist' \
  "$LOCAL_DIR/" "$REMOTE_HOST:$REMOTE_DIR/"

echo "✅ 首次同步完成，开始监听文件变化..."

# 实时监听并同步
fswatch -o "$LOCAL_DIR" | while read f; do
  echo "📝 检测到变更，正在同步... $(date '+%H:%M:%S')"
  rsync -avz --delete \
    --exclude='.git' \
    --exclude='node_modules' \
    --exclude='.DS_Store' \
    --exclude='dist' \
    "$LOCAL_DIR/" "$REMOTE_HOST:$REMOTE_DIR/"
  echo "✅ 同步完成"
done
```

**使用方式**：
```bash
# 1. 终端1：启动实时同步（保持运行）
./sync.sh

# 2. 终端2：在服务器上构建并运行（开发调试）
ssh zhou@192.168.31.188
cd ~/projects/{project}
npm run build
docker build ...
```

**优点**：
- 修改代码后1秒内同步到服务器
- 无需手动触发部署
- 适合高频修改的开发阶段

**缺点**：
- 需要保持终端运行
- 只同步代码，不自动构建

---

### 模式B：完整部署发布（推荐发布/更新使用）

**脚本**：`deploy188.sh`

**适用场景**：
- 功能开发完成，正式部署
- 阶段性发布
- CI/CD 自动化部署

**机制**：
```
本地代码
    ↓
rsync 同步到服务器 /opt/{project}-build/
    ↓
服务器端 npm install
    ↓
服务器端 npm run build
    ↓
服务器端 docker build
    ↓
docker compose up -d
```

**使用方式**：
```bash
./deploy188.sh
```

**优点**：
- 一次命令完成全部流程
- 服务器端构建，避免跨平台问题
- 完整的部署日志

**缺点**：
- 每次修改需要重新运行
- 耗时较长（构建需要时间）

## 模式对比与选择

| 维度 | sync.sh (实时同步) | deploy188.sh (完整部署) |
|------|-------------------|------------------------|
| **运行方式** | 持续运行 | 一次性 |
| **同步延迟** | 1秒内 | 手动触发 |
| **是否构建** | 否（需手动） | 是（自动） |
| **适用阶段** | 开发阶段 | 发布阶段 |
| **使用频率** | 高频（每次修改） | 低频（每个版本） |
| **目标目录** | `~/projects/` | `/opt/{project}-build/` |

### 推荐工作流

**开发阶段**：
```bash
# 终端1：持续同步
./sync.sh

# 终端2：服务器上开发调试
ssh zhou@192.168.31.188
cd ~/projects/{project}/apps/frontend
# 修改代码 → 自动同步 → 刷新查看效果
```

**发布阶段**：
```bash
# 功能开发完成，完整部署
./deploy188.sh
```
